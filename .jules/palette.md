## 2025-02-18 - Confirmation Step in Destructive CLI Commands
**Learning:** Users appreciate a pause for confirmation in long-running or destructive CLI processes, especially when presented with a diff or changes to review. Without it, the review step is useless as the process continues immediately.
**Action:** Always include a confirmation prompt (e.g., "Press Enter to continue") after displaying information that the user is expected to review before a critical action. Use color (e.g., yellow) to draw attention to the prompt.
